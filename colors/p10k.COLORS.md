# p10k.zsh color customization

`p10k.zsh.symlink` is generated by `p10k configure`. The wizard writes
xterm-256 color codes (`POWERLEVEL9K_FOO_FOREGROUND=178`). We replace all
those with hex colors drawn from the **LSD Warm Light** palette, routed
through an associative array near the top of the file so the theme can be
retuned by editing a single table.

## The palette

Defined inside the anonymous function near the top of `p10k.zsh.symlink`,
right after the `Zsh >= 5.1` version check:

```zsh
local -A clr=(
  fg              '#1e1408'   # base text (deep brown)
  fg_muted        '#4a3a20'   # softer text
  fg_dim          '#6a5a40'   # ruler, multiline gap
  fg_faint        '#8a7a55'   # shortened paths, low-emphasis
  accent          '#c47a14'   # signature warm amber
  accent_bright   '#d4a054'
  red             '#d42020'
  red_bright      '#e03030'
  green           '#2e7d16'
  green_bright    '#378a20'
  yellow          '#a06800'
  yellow_bright   '#b87800'
  blue            '#1a60c0'
  blue_bright     '#2570d0'
  magenta         '#8b2ab8'
  magenta_bright  '#a035d0'
  cyan            '#0e7e7e'
  cyan_bright     '#1a8e8e'
  orange          '#c45a00'
)
```
        
These hex values come from the canonical palette in
`colors/lsd-colors.html`, also mirrored in `zed/lsd-warm.json.symlink`'s
`terminal.ansi.*` entries (light theme) and in
`ghostty/LSD-Warm-Light.symlink`.

## How references look

Every `POWERLEVEL9K_*_FOREGROUND` assignment uses `${clr[name]}`:

```zsh
typeset -g POWERLEVEL9K_DIR_FOREGROUND=${clr[blue]}
typeset -g POWERLEVEL9K_PROMPT_CHAR_OK_{VIINS,VICMD,VIVIS,VIOWR}_FOREGROUND=${clr[green_bright]}
typeset -g POWERLEVEL9K_STATUS_ERROR_FOREGROUND=${clr[red]}
```

p10k accepts hex colors of the form `#rrggbb` directly. Brace expansion
still works â€” `{A,B}=value` becomes `A=value B=value`, and `${clr[x]}` is
substituted at runtime.

## Re-applying after `p10k configure`

The wizard overwrites `p10k.zsh.symlink` with a fresh file full of
numeric color codes. To restore the warm palette:

1. **Insert the `clr` table** after the `Zsh >= 5.1` check, before the
   first `POWERLEVEL9K_*` assignment. Copy it verbatim from the section
   above (or from git history of this file).

2. **Replace every numeric foreground.** With Claude Code, the fastest
   path is a series of file-wide replacements:

   ```
   _FOREGROUND=242  â†’  _FOREGROUND=${clr[fg_dim]}
   _FOREGROUND=76   â†’  _FOREGROUND=${clr[green_bright]}
   _FOREGROUND=196  â†’  _FOREGROUND=${clr[red_bright]}
   ... etc per the mapping table above ...
   ```

   The wizard may emit color codes we haven't seen before. For each
   unknown code, look it up in `xterm-256`
   (`for i in {0..255}; do print -Pn "%K{$i}  %k%F{$i}${(l:3::0:)i}%f " ${${(M)$((i%6)):#3}:+$'\n'}; done`),
   identify the closest hue family in the `clr` table, and add it to
   the mapping.

3. **Verify**:

   ```
   grep -nE '_FOREGROUND=[0-9]' oh-my-zsh/p10k.zsh.symlink   # should output nothing
   zsh -n oh-my-zsh/p10k.zsh.symlink                          # syntax check
   zsh -c 'source ~/.zshrc && echo OK'                        # functional check
   ```

## Adjusting the palette

To shift the whole theme (e.g., to a Warm Dark or a cooler variant),
edit the `clr` table only. Every segment picks up the new hue
automatically. Each hex value can be:

- A real hex literal (`#1a60c0`) â€” preferred.
- An xterm-256 code as a string (`'31'`) â€” also accepted by p10k, useful
  if you want a hue to fall back to terminal-theme ANSI colors instead
  of being pinned to one hex value.

## Background colors

`POWERLEVEL9K_BACKGROUND=` (line ~127) is left empty so the prompt is
transparent â€” terminal background shows through, which lets the same
prompt look good against both Warm Light and Warm Dark ghostty themes.
No per-segment backgrounds are set.
